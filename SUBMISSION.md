# Model Submission Guide

Leaderboard submissions should include:

- Model ID and revision.
- Model type: `open-weight`, `open-source`, or `api`.
- Backend: `hf` or `openai-compatible`.
- Exact command used for evaluation.
- Result JSON generated by `persian-eval`.
- License and model card link.
- Hardware/backend notes if they affect reproducibility.

Example open-weight run:

```bash
persian-eval run \
  --model org/model-name \
  --backend hf \
  --model-type open-weight \
  --revision <commit-or-tag> \
  --data data/persian_eval_v1.public_eval.jsonl \
  --output results/org__model-name.json
```

Before submitting:

```bash
persian-eval validate results/org__model-name.json
```

API baselines are accepted only as reference rows and should not be mixed into the main open-weight ranking.

For large models on 24GB GPUs, use explicit quantization metadata in the command and result:

```bash
persian-eval run \
  --model org/model-14b \
  --backend hf \
  --model-type open-weight \
  --quantization 4bit \
  --dtype bfloat16 \
  --data data/persian_eval_v1.hard.jsonl \
  --output results/org__model-14b_4bit_hard.json
```
